## СЕССИЯ 29.03.2026 — БАГФИКСЫ И УЛУЧШЕНИЯ

### Проблема 1: Модалки закрываются при клике на фон
**Исправлено.** Убран onClick={onClose} с overlay в:
- `ClientForm.jsx`
- `ContractForm.jsx`
- `TaskForm.jsx`
Теперь модалки закрываются только по крестику или кнопке "Отмена".

### Проблема 2: Дубликаты при импорте Excel
**Исправлено.**
- `importService.js` — нормализация имён: убирает `[12345]` (ID из другой системы), кавычки «»"", лишние пробелы. Автоопределение колонки ФИО по содержимому (если заголовок пустой). Предзагрузка всех клиентов в кэш для быстрого сравнения. Swap дат если endDate < startDate.
- `clientService.js` — добавлены `findDuplicates()` (группировка по нормализованному имени) и `mergeClients()` (перенос всех договоров/задач/заметок + удаление дубликата).
- `clients.js route` — новые эндпоинты `GET /api/v1/clients/duplicates` и `POST /api/v1/clients/merge`.
- `Clients.jsx` — баннер "Найдено N групп дубликатов" + модалка для просмотра и объединения. Кнопки "Объединить" и "Пропустить" для каждой группы.

### Проблема 3: Фильтр по датам в договорах
**Исправлено.**
- `contractService.js` — поддержка `dateFrom`/`dateTo` параметров для фильтрации по дате начала.
- `contracts.js route` — пробрасывает `dateFrom`/`dateTo`.
- `Contracts.jsx` — два поля "Период: С — По" в тулбаре + кнопка "Сбросить фильтры" когда есть активные фильтры.
- Новые CSS стили в `client/src/styles/date-filter.css`.

### Проблема 4: Акты — улучшенная сверка + начисленное КВ
**Исправлено.**
- `Contract.js model` — новое поле `accrualCommission` (Number, default: 0). Накопительное — каждый акт прибавляет/вычитает.
- `actService.js` — 4-уровневый нечёткий поиск:
  1. Точное совпадение номера
  2. Частичное совпадение номера (один содержит другой)
  3. Имя клиента + частичный номер
  4. Только имя клиента (если один договор)
  Каждое совпадение помечается confidence: exact/partial/name_partial/name_only/none.
  Новая функция `updateAccrualCommissions()` — обновляет `$inc: { accrualCommission: actualAmount }` на найденных договорах.
- `acts.js route` — после сохранения акта вызывает `updateAccrualCommissions`.
- `Contracts.jsx` — новая колонка "КВ начисл." в таблице и мобильных карточках. В итогах показывается `totalAccrualCommission`.
