## СЕССИЯ 29.03.2026 (ВЕЧЕР) — НЕЧЁТКИЕ ДУБЛИКАТЫ, АКТЫ, ПЕРФОРМАНС

### 1. Нечёткий поиск дубликатов клиентов (3 уровня)
**Файл:** `server/src/services/clientService.js`
- **Уровень 1+2 (deep):** Убирает юр. формы (ООО, ОАО, ЗАО, АО, ПАО, ИП, МБУ, ГБУСО, ГБУ, МУП, МКУ, ФГУП, ФГБУ, ГУП, НКО, АНКО, ТСЖ, СЗ). Замена ё→е. Убирает ВСЕ пробелы/точки/дефисы. **Также убирает юр. формы как префикс** (ОООБЛОКСТРОЙ → блокстрой).
- **Уровень 3 (abbreviation):** "Петров А.И." совпадает с "Петров Александр Иванович" по фамилии + инициалам.
- **Защита от тёзок:** Если у обоих клиентов есть ДР и они разные → НЕ предлагает объединять.

### 2. Защита от тёзок при импорте
**Файл:** `server/src/services/importService.js`
- Кэш хранит массив клиентов на одно имя (для тёзок).
- Если у строки есть ДР и у найденного клиента другой ДР → создаёт нового клиента.
- Проверка дубликатов: **номер + СК + премия + даты + тип** (не теряет пролонгации и разные типы).

### 3. Объединение контактов при merge через запятую
**Файл:** `server/src/services/clientService.js`
- Функция `mergeField()` — если у обоих есть телефон, будет "111, 222". То же с email и заметками.

### 4. Акты — кнопка "Добавить договор" для ненайденных
**Файл:** `server/src/routes/acts.js`
- Новый эндпоинт `POST /api/v1/acts/:id/add-contract` — создаёт клиента+договор из строки акта.
- Использует `normalizeNameForCompare` из importService для поиска существующего клиента (не создаёт дубликат).
- После добавления обновляет статус строки акта на "found".

**Файл:** `client/src/pages/Acts.jsx`
- Статус "Не найден" → "Договор не найден".
- Кнопка **"+ Добавить"** рядом с каждой строкой "Договор не найден".
- Сводка: "N найдено / M не найд."
- CSS: `client/src/styles/accrual.css`

### 5. Модалки — закрытие по клику на фон убрано ВЕЗДЕ
Исправлены ВСЕ модалки (включая пропущенные ранее):
- `ImportModal.jsx` — убран onClick={onClose} с overlay
- `ClientDetail.jsx` — модалка "Ссылка на документы" — убран onClick

### 6. Фильтр по периоду — пересечение диапазонов
**Файл:** `server/src/services/contractService.js`
- Фильтрует по пересечению: `startDate <= dateTo AND (endDate >= dateFrom OR endDate is null)`.

### 7. Сортировка по имени клиента в договорах
**Файл:** `server/src/services/contractService.js` + `client/src/pages/Contracts.jsx`
- Колонка "Клиент" теперь сортируемая (in-memory sort после populate с `localeCompare('ru')`).

### 8. КВ начисленное — убрана отдельная колонка
**Файл:** `client/src/pages/Contracts.jsx`
- Колонка "КВ начисл." убрана. Начисленное показывается под ожидаемым в той же ячейке "КВ" мелким жёлтым шрифтом.
- Таблица вернулась к 10 колонкам.

### 9. Откат accrualCommission при удалении акта
**Файл:** `server/src/routes/acts.js`
- При DELETE акта вычитаются суммы обратно из accrualCommission на договорах.

### 10. Производительность дашборда
**Файл:** `server/src/services/dashboardService.js`
- `getByCompany`, `getByType`, `getStats` — переведены на MongoDB **aggregation** (было ~1200ms → ~50-100ms).
- `getAlerts` — загружает только нужные контракты (истекающие + с неоплаченными взносами), `.lean()`.

**Файл:** `server/src/app.js`
- Добавлено `app.set('trust proxy', 1)` — фикс ошибки rate-limit за nginx.

### 11. Модель Act — добавлено поле `found`
**Файл:** `server/src/models/Act.js`
- Статус enum: `['ok', 'diff', 'found', 'unknown']`
- Новые поля: `confidence`, `contractId`
