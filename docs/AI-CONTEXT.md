## АУДИТ КОДА (23.03.2026)

**Общая оценка: 8/10** (после исправлений)

### Исправлено в этой сессии:
- **POST /auth/refresh** — endpoint для обновления access token через refresh cookie. Раньше пользователя выкидывало через 15 минут. Теперь автообновление незаметно, разлогин только после 7 дней неактивности.
- **POST /auth/logout** — очищает refresh cookie на сервере.
- **api.js** — автоматический refresh при 401: перехватывает ошибку, вызывает /auth/refresh, повторяет запрос. Очередь запросов при одновременных 401.
- **AuthContext.jsx** — logout вызывает серверный endpoint.
- **Серверная фильтрация по clientId** — contracts и tasks API поддерживают `?clientId=xxx`. ClientDetail.jsx больше не загружает ВСЕ договоры/задачи и не фильтрует на клиенте.

### Безопасность (8.5/10):
- Пароли: bcrypt salt 10 ✓
- JWT: access 15мин + refresh 7д в httpOnly cookie + автообновление ✓
- MongoDB: localhost only + пароль + фаервол (3 уровня) ✓
- Helmet, mongo-sanitize, CORS, rate limiting ✓
- Данные изолированы по userId ✓
- Ошибки не раскрывают внутреннюю информацию ✓

### Некритичные улучшения (на будущее):
- Разные JWT секреты для access/refresh
- Логирование неудачных входов
- TypeScript (при команде 3+)
- Вынести formatDate/formatCurrency в utils/
- Разбить ClientDetail.jsx на под-компоненты
- Серверный endpoint для календаря (/calendar/events?month=)
- Redis для кеширования дашборда
- MongoDB aggregation вместо find+loop в getTotals
