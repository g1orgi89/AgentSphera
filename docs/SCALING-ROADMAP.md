# SCALING-ROADMAP — АгентСфера

> План масштабирования: от одного агента к SaaS-продукту для индивидуальных страховых агентов.
> Каждый этап — самостоятельная ценность. Не нужно делать всё сразу.

---

## ТЕКУЩЕЕ СОСТОЯНИЕ

**Что уже есть:**
- Полный CRM-функционал (клиенты, договоры, взносы, комиссии, задачи, календарь, акты, дашборд)
- Изоляция данных по userId — каждый агент видит только своё
- Регистрация и авторизация (JWT + refresh token)
- Мобильная адаптация + PWA
- Импорт/экспорт Excel
- AI-парсинг актов (Anthropic Claude)
- VPS с MongoDB, nginx, SSL

**Что нужно для продажи:**
- Система тарифов и оплаты
- Верификация пользователей
- Юридическая база
- Админ-панель
- Мониторинг и стабильность

---

## ЭТАП 1 — ГОТОВНОСТЬ К ПЕРВЫМ ПОЛЬЗОВАТЕЛЯМ

> Цель: 10-50 агентов могут безопасно пользоваться системой
> Срок: 1-2 недели

### 1.1 Подтверждение email при регистрации
- Отправка кода/ссылки на email после регистрации
- Аккаунт неактивен до подтверждения
- Пакет: nodemailer + SMTP (Яндекс, Mail.ru или SendGrid)
- Файлы: `server/src/services/emailService.js`, обновить `routes/auth.js`, модель User (+emailVerified, +verificationToken)

### 1.2 Восстановление пароля
- «Забыли пароль?» на странице логина
- Email с ссылкой/кодом для сброса
- Файлы: `routes/auth.js` (POST /auth/forgot, POST /auth/reset), обновить Login.jsx

### 1.3 Автотесты бэкенда (задача 6.5)
- Jest + Supertest + mongodb-memory-server
- 5 групп: авторизация, клиенты, договоры, акты, каскадное удаление
- ~15-20 тестов, ~200 строк
- Файлы: `server/tests/`

### 1.4 Бэкапы MongoDB
- Автоматический mongodump по cron (ежедневно, хранить 30 дней)
- Скрипт: `server/scripts/backup.sh`
- Cron: `0 3 * * * /var/www/agentsfera/server/scripts/backup.sh`

### 1.5 SSH безопасность
- Пользователь `deploy` вместо root
- SSH по ключу, отключить вход по паролю
- PermitRootLogin no

### 1.6 Мониторинг
- UptimeRobot (бесплатно) — проверяет доступность каждые 5 мин, уведомления в Telegram
- PM2 Plus или pm2-logrotate для ротации логов

### 1.7 Юридика (минимум для старта)
- Страница «Пользовательское соглашение» (оферта)
- Страница «Политика конфиденциальности» (152-ФЗ)
- Согласие при регистрации (чекбокс)
- Файлы: `client/src/pages/Terms.jsx`, `Privacy.jsx`

## ЭТАП 2 — МОНЕТИЗАЦИЯ

> Цель: агенты платят за подписку
> Срок: 2-3 недели

### 2.1 Модель тарифов
- **Бесплатный:** до 10 клиентов, до 20 договоров, без AI-парсинга актов
- **Базовый (~500₽/мес):** до 100 клиентов, безлимит договоров, AI-парсинг
- **Про (~1500₽/мес):** безлимит, приоритетная поддержка, экспорт отчётов
- Файлы: модель Subscription, middleware checkLimits

### 2.2 Интеграция оплаты
- ЮKassa (для РФ) или CloudPayments
- Рекуррентные платежи (автосписание)
- Вебхуки для подтверждения оплаты
- Файлы: `server/src/routes/billing.js`, `server/src/services/billingService.js`
- Пакет: @yookassa/sdk или cloudpayments

### 2.3 Лимиты по тарифу
- Middleware `checkLimits` — проверяет количество клиентов/договоров перед созданием
- Красивое сообщение «Лимит достигнут. Перейдите на Базовый тариф»
- Файлы: `server/src/middleware/limits.js`

### 2.4 Страница тарифов
- На лендинге: сравнение тарифов
- В приложении: текущий тариф, кнопка «Улучшить»
- Файлы: обновить Landing.jsx, добавить `client/src/pages/Billing.jsx`

### 2.5 Пробный период
- 14 дней бесплатно на тарифе «Базовый» после регистрации
- После окончания — переход на бесплатный или оплата

## ЭТАП 3 — РОСТ И УДЕРЖАНИЕ

> Цель: 100-500 агентов, стабильная выручка
> Срок: 1-2 месяца

### 3.1 Админ-панель
- Отдельный раздел для role: 'admin'
- Список пользователей, их активность (последний вход, кол-во клиентов/договоров)
- Управление тарифами, ручная активация
- Статистика: выручка, регистрации, отток
- Файлы: `client/src/pages/Admin.jsx`, `server/src/routes/admin.js`

### 3.2 Онбординг нового пользователя
- Приветственный тур (3-4 шага): «Добавьте первого клиента → Создайте договор → Настройте задачу»
- Пустые состояния с полезными подсказками (уже частично есть)
- Демо-данные: кнопка «Заполнить демо-данные» для знакомства

### 3.3 Уведомления
- Email-уведомления: истекающие договоры, просроченные взносы, задачи на завтра
- Настройки уведомлений в профиле (вкл/выкл, частота)
- Пакет: nodemailer + cron
- Файлы: `server/src/services/notificationService.js`, `server/src/jobs/notifications.js`

### 3.4 Улучшение лендинга
- Блок «Тарифы» с ценами
- Блок «Отзывы» (реальные от первых пользователей)
- Блок «Как это работает» (3 шага с скриншотами)
- Блок «FAQ»
- SEO: meta-теги, Open Graph, sitemap.xml

### 3.5 Производительность
- MongoDB aggregation вместо find+loop в getTotals и dashboardService
- Серверный endpoint для календаря с фильтром по месяцу
- Индексы: проверить и добавить для частых запросов
- PM2 cluster mode (2 инстанса на 2 CPU)

### 3.6 Telegram-бот для поддержки
- Бот принимает вопросы от пользователей
- Уведомления администратору о новых тикетах
- Или простой email: support@агентсфера.рф

## ЭТАП 4 — МАСШТАБИРОВАНИЕ ИНФРАСТРУКТУРЫ

> Цель: 500+ агентов, высокая нагрузка
> Срок: по необходимости

### 4.1 Redis
- Кеширование дашборда (инвалидация при изменении данных)
- Хранение сессий
- Rate limiting через Redis (вместо in-memory)

### 4.2 Миграция MongoDB
- MongoDB Atlas (облако) — автобэкапы, мониторинг, масштабирование
- Или реплика-сет на отдельном сервере

### 4.3 CDN для статики
- Cloudflare (бесплатный тариф) — кеширование статики, защита от DDoS
- Или Selectel CDN для РФ

### 4.4 CI/CD
- GitHub Actions: автотесты при push, автодеплой при merge в main
- Staging-среда для тестирования перед продакшеном

### 4.5 Логирование и мониторинг
- Grafana + Prometheus или Sentry для ошибок
- Алерты при росте ошибок, падении сервера

### 4.6 Горизонтальное масштабирование
- Несколько VPS за балансировщиком (если >1000 пользователей одновременно)
- Stateless архитектура (токены в Redis, файлы в S3)

## ЭТАП 5 — АГЕНТСТВА (B2B)

> Цель: продажа не только агентам, но и страховым агентствам
> Срок: когда будет product-market fit с агентами

### 5.1 Мультитенантность
- Модель Organization (агентство)
- Агент привязан к организации
- Руководитель видит данные всех агентов организации
- Дашборд агентства: общая аналитика, сравнение агентов

### 5.2 Роли внутри организации
- Владелец (полный доступ)
- Менеджер (видит всех агентов, не может удалять)
- Агент (видит только своё)

### 5.3 Отчёты для руководителя
- Сводка по агентам: кто сколько собрал, КВ, задолженность
- Экспорт отчётов в Excel/PDF
- Графики: динамика по месяцам, сравнение агентов

### 5.4 Тарифы для агентств
- Цена за агента (например 300₽/мес за агента)
- Скидка за объём (10+ агентов — дешевле)
